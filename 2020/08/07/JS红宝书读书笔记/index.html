JS红宝书读书笔记

第一章 JavaScript简介

JavaScript实现由三部分组成：

  1. 核心（ECMAScript）
  2. 文档对象模型（DOM）
  3. 浏览器对象模型（BOM）

DOM

DOM 1级：

  • DOM核心：规定如何映射基于XML的文档解构，以便简化对文档中任意部分的访问和操作
  • DOM HTML：在DOM核心的基础上加以扩展，添加了针对HTML的对象和方法

DOM 2级：

  • DOM视图
  • DOM事件
  • DOM样式
  • DOM遍历和范围

DOM 3级：

  • DOM加载和保存
  • DOM验证
  • 扩展DOM核心

BOM

  • window
  • navigator
  • location
  • screen
  • 对cookie的支持
  • XMLHttpRequest和IE的ActiveXObject这样的自定义对象

第二章 在HTML中使用JS

<script>

  • async：
    • 立即异步下载脚本并执行
    • 不保证执行顺序
    • 一定会在load事件前执行，但可能会在DOMContentLoaded事件触发之前或之后执行
  • defer：
    • 立即下载，但延迟执行。延迟到浏览器遇到</html>标签后再执行。
    • 不一定会按照顺序执行，也不一定会在DOMContentLoaded事件触发前执行，所以最好只包含一个延迟脚本。
    • 一些浏览器会忽略这个属性，所以把延迟脚本放在页面底部依旧是最佳选择。
  • src：在带有src属性的script元素中的js代码会被忽略。

文档模式

  • 混杂模式
  • 标准模式

第三章 基本概念

标识符

  • 第一个字符必须是一个字母、_ 或 $
  • 其他字符可以是字母、下划线、美元符号或数字
  • 不能把关键字、保留字、true、false和null用作标识符

严格模式

​ “use strict”

变量

用var操作符定义的变量将成为定义该变量的作用域的局部变量。

省略var定义全局变量

数据类型

6种简单数据类型、1种复杂数据类型

type of

可以检测的类型：undefined、boolean、string、number、object、function

是操作符不是函数

typeof null —— “object”

Undefined

对于未初始化或是未声明的元素，执行typeof都为undefined

所以最好在声明的时候就初始化，以防出现undefined，无法辨别是没有声明还是没有初始化

Null

null表示一个空对象指针，所以typeof null是“object”

如果定义的变量将来用于保存对象，那么最好初始化未null

undefined 派生自 null

null == undefined // true

Boolean

数据类型 true false
String 非空字符串 “”
Number 任何非零数字值（包括无穷大） 0和NaN
Object 任何对象 null
Undefined n/a undefined

Number

  • 进制
    • 八进制：第一位必须是0，然后是0~7。如果数值超出范围，前导0会被忽略，当作十进制解析。
    • 十六进制：前两位为0x， 然后是09、AF。
  • 浮点数：
    • 浮点数需要的内存空间是保存整数的两倍，所以ES会不失时机地将浮点数值转换为整数数值。
    • ES会将小数点后带有6个零以上的浮点数值转换为e表示法表示的数值

浮点数值最高精度是17位小数，0.1+0.2不等于0.3，而是0.3000000000000004

  • 数值范围

    • Number.MAX_VALUE、Number.MIN_VALUE，超过界限自动转换为Infinity
    • isFinite()可以判断是否为Infinity
    • Number.NEGATIVE_INFINITY、Number.POSITIVE_INFINITY也可以得到正负Infinity的值

  • NaN：

    • 任何关于NaN的操作都会返回NaN
    • NaN和任何值都不想等，包括他自己
    • 所以有了isNaN()

  • 数值转换：

    • Number() 可以用于任何数据类型

    • parseInt()、parseFloat() 专门用于字符串
      • parseInt(字符串，进制)
      • parseFloat(字符串) 只用于十进制
